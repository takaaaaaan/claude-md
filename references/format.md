# 正規 CLAUDE.md 形式（House Style）

`claude-md-init` と `claude-md-align` の **唯一の正本**。両スキルはこのファイルを読み、ここで定義した骨格に従う。骨格を変えたいときは **このファイルだけ** を直す。

逐語で焼き込むテキスト（A ブロックの本文）は [`fixed-blocks.md`](./fixed-blocks.md) にある。

| 12 | 姉妹 / 関連リポ説明 | **C 条件付き** | FE↔BE のような相互参照がある時だけ。相手の構造を Q&A 形式で載せる |

### 区分の意味

- **A 固定**: あなたの流儀。プロジェクトに関係なく毎回ほぼ逐語で入る（`fixed-blocks.md`）。`align` では欠けていたら補い、`init` では必ず入れる。
- **B 生成**: コードベースを解析して中身を書く。`align` では **既存の中身を最大限保全** し、空欄や明確な実態ズレだけ直す。
- **C 条件付き**: 該当する状況（別リポと密結合）でのみ追加。無ければセクションごと省く。

## B セクションを生成するための解析チェックリスト（init 用）

コードベースから以下を読み取って B セクションを埋める。`/init` と同じ要領。

1. **スタック検出** — `package.json` / `pyproject.toml` / `requirements.txt` / `go.mod` / `Cargo.toml` / `Dockerfile` / `docker-compose.yml` / lockfile からパッケージマネージャ・言語・フレームワークを判定。
2. **コマンド抽出** — `package.json` の `scripts`、`Makefile`、`pyproject.toml` のツール設定、README の手順から **setup / 起動 / テスト** を拾う。実在するものだけ載せる(でっち上げない)。
3. **ポート検出** — `docker-compose.yml`・`next.config`・`uvicorn`/`vite`/`dev` の既定ポートを拾い、セクション 5 の「デフォルトポート」「予約レンジ」を実値で埋める。検出できなければ user に確認。
4. **ディレクトリ構成** — トップ〜2 階層のツリーを取り、各ディレクトリの責務を 1 行で注釈（セクション 6）。
5. **アーキテクチャの核** — エントリポイント、データ層、状態管理、主要なフロー/スキーマ、外部サービス連携を特定（セクション 7）。
6. **拡張指針** — 繰り返し現れるパターン(アダプタ追加・ルート追加・スキーマ変更など)を見つけ「X を足すなら Y」に言い換える（セクション 8）。
7. **姉妹リポ判定** — README / import / `.gitmodules` / 兄弟ディレクトリから別リポ依存を検出。あれば C を追加し、無ければ user に「関連リポはあるか」を確認。

**捏造しないルール**: 確認できない事実（存在しないコマンド・ポート・ファイル）は書かない。不明点は user に質問するか、`<!-- TODO: 要確認 -->` を残す。
